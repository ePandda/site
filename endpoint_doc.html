Endpoints

The ePandda public API supports HTTP GET search requests for the following endpoints: Publications, Occurrences and Geonames.

The API is a RESTful web service that delivers data as JSON documents. All GET requests must be URL-encoded.

lithostratigraphy

Returns specimens matching lithostratigraphic units

Name Type Required? Description
lithostratigraphy text False Search on the full lithostratigraphic hierarchy
rank text False Limit your search to a lithostratigraphic rank. Accepted values are: Bed, Member, Formation & Group

publications

Returns specimen and publication records for a given scientific name. Results may be filtered using the available parameters.

Name Type Required? Description
scientific_name text False Taxon to search occurrence records for
journal text False Then name of academic Journal a publication would be found
article text False The name of the journal article the given scientific_name appears in
author text False The name of the author who's article describes the given scientific_name
stateProvinceName text False The state/province to search for scientific_name and publication references
county text False The county to search for scientific_name and publication references
locality text False The locality name to search for scientific_name occurences and publication references
includeAnnotations boolean False Toggles if OpenAnnotations section should be included or not

stratigraphy

Returns specimens matchting chronostratigraphic terms

Name Type Required? Description
chronostratigraphy text False Search for a single term in the chronostratigraphic hierarchy
stage text False Search a single or range of age/stages. Can be formatted as 'age1-age2' or 'age1 to age2'
series text False Search a single or range of series/epochs. Can be formatted as 'epoch1-epoch2' or 'epoch1 to epoch2'
system text False Search a single or range of age/stages. Can be formatted as 'period1-period2' or 'period1 to period2'
erathem text False Search a single or range of age/stages. Can be formatted as 'era1-era2' or 'era1 to era2'

occurrences

Returns specimens collected from a given locality

Name Type Required? Description
taxon_name text False The taxa to search occurrences for
locality text False The locality name to bound taxonomic occurences to
chronostratigraphy text False The geologic time period to filter taxon occurrences by
lithostratigraphy text False The lithostratigraphic unit to filter taxon occurrences by

annotations

Returns openAnnotations for linked data in ePANDDA.

Name Type Required? Description
annotationDate text False Filter annotation results by provided date ( simple date match ). Format annotationDate=YYYY-MM-DD
annotationDateAfter text False Filter annotation results equal to or after provided date. Format annotationDateAfter=YYYY-MM-DD
annotationDateBefore text False Filter annotation results before provided date. Format annotationDateBefore=YYYY-MM-DD

geonames

Returns specimens collected from a given locality

Name Type Required? Description
geolocation text False A generic place name, will search across locality, county, state/province and country fields
countryName text False Full country names
countryCode text False 2-letter country code, provided from geonames.org
stateProvinceName text False State or province name. Do not include qualifiers (e.g. State, Province) to return better results
stateProvinceCode text False State or province name. Do not include qualifiers (e.g. State, Province) to return better results
county text False County name. Do not add "County" to your search string
locality text False The specific locality where a specimen was recorded
geoPoint text False The Latitude, Longitude to search on. Provide a comma separated string with no space (e.g. '-103.2,29.2').
geoRadius text False The distance from the provided geoPoint to search within, in meters. Provide only the distance with no units

taxonomy

Returns specimens matchting taxonomic terms

Name Type Required? Description
fullTaxonomy text False A search across all levels of of the taxonomic hierarchy
genus text False Genus
species text False The species (specific epithet) of the specimen
scientificName text False A formatted scientific name, including or not including scientific name authorship and year
family text False Taxonomic family
order text False Taxonomic order
class text False Taxonomic class
phylum text False Taxonomic phylum
kingdom text False Taxonomic kingdom
other text False Other terms such as superFamily, 'above family' as well as a generic 'noRank' field
images boolean False Set to true to return images any images from iDigBio that match the search terms

stats

Interesting API statistics

Name Type Required? Description
stateProvinces boolean False The number of unique states represented in the collections
countries boolean False The number of unique countries represented in the collections
counties boolean False The number of unique counties represented in the collections
localities boolean False The number of unique localities represented in the collections
geoPoints boolean False The number of unique geographic coordinates represented in the collections
totalRecords boolean False The count of all specimen/occurrence records included in project from iDigBio and PBDB
taxonomies boolean False The number of unique taxonomic hierarchies represented in the collections
