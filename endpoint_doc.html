Endpoints

The ePandda public API supports HTTP GET search requests for the following endpoints: Publications, Occurrences and Geonames.

The API is a RESTful web service that delivers data as JSON documents. All GET requests must be URL-encoded.

publications

Returns specimen occurrence and publication records for a given scientific name. Results may be filtered using the available parameters.

Name Label Type Required? Description
scientific_name Scientific Name text True Taxon to search occurrence records for
journal Journal text False Then name of academic Journal a publication would be found
article Article text False The name of the journal article the given scientific_name appears in
author Author text False The name of the author who's article describes the given scientific_name
state_province State/Province text False The state/province to search for scientific_name and publication references
county County text False The county to search for scientific_name and publication references
locality Locality text False The locality name to search for scientific_name occurences and publication references

stratigraphy

Returns specimens matchting chronostratigraphic terms

Name Label Type Required? Description
chronostratigraphy Chronostratigraphy Full Text text False Search on the a full chronostratigraphic hierarchy
stage Stage/Age text False The chronostratigraphic Stage/Age. Searches both high/low
series Series/Epoch text False The chronostratigraphic Series/Epoch. Searches both high/low
system System/Period text False The chronostratigraphic System/Period. Searches both high/low
erathem Erathem/Era text False The chronostratigraphic Erathem/Era. Searches both high/low

occurrences

Returns specimens collected from a given locality

Name Label Type Required? Description
taxon_name Taxonomy text False The taxa to search occurrences for
locality Locality text False The locality name to bound taxonomic occurences to
chronostratigraphy Chronostratigraphy text False The geologic time period to filter taxon occurrences by

geonames

Returns specimens collected from a given locality

Name Label Type Required? Description
geolocation Full Text Geographic Search text False A generic place name, will search across locality, county, state/province and country fields
countryName Country Name text False Full country names
countryCode Country Code (2-Letter) text False 2-letter country code, provided from geonames.org
stateProvinceName State/Province Name text False State or province name. Do not include qualifiers (e.g. State, Province) to return better results
stateProvinceCode State Province Code (2-Letter or 2-Digit) text False State or province name. Do not include qualifiers (e.g. State, Province) to return better results
county County Name text False County name. Do not add "County" to your search string
locality Locality Name text False The specific locality where a specimen was recorded
geoPoint Geographic Point (Latitude, Longitude) text False The Latitude, Longitude to search on. Provide a comma separated string with no space (e.g. '-103.2,29.2').
geoRadius Distance from Geographic Point in Meters text False The distance from the provided geoPoint to search within, in meters. Provide only the distance with no units

taxonomy

Returns specimens matchting taxonomic terms

Name Label Type Required? Description
fullTaxonomy Taxonomy Full Text text False A search across all levels of of the taxonomic hierarchy
scientificName Scientific Name text False A specific scientific name, including or not including scientific name authorship and year
species Species text False The species (specific epithet) of the specimen
genus Genus text False Genus name
family Family text False Family name
order Order text False State or province name. Do not include qualifiers (e.g. State, Province) to return better results
class Class text False Class name
phylum Phylum text False The specific locality where a specimen was recorded
kingdom Kingdom text False The Longitude, Latitude to search on. Provide a comma separated string. IMPORTANT geoJSON requries LONGITUDE first.
other Other JSON False Other terms such as superFamily, 'above family' as well as a generic 'noRank' field
images Taxon Images boolean False Set to true to return images any images from iDigBio that match the search terms

stats

Interesting API statistics

Name Label Type Required? Description
stateProvinces States/Provinces boolean False The number of unique states represented in the collections
countries Countries boolean False The number of unique countries represented in the collections
counties Counties boolean False The number of unique counties represented in the collections
localities Localities boolean False The number of unique localities represented in the collections
geoPoints Geographic Coordinates boolean False The number of unique geographic coordinates represented in the collections
totalRecords Total Records boolean False The count of all specimen/occurrence records included in project from iDigBio and PBDB
taxonomies Taxonomies boolean False The number of unique taxonomic hierarchies represented in the collections
